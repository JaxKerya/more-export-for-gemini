# Changelog

All notable changes to **More Export for Gemini** are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Fixed
- **Documentation matched to reality (store-review risk):** `PRIVACY.md` and `PROJECT_SUMMARY.md` listed formats that no longer exist (ODT, reStructuredText, AsciiDoc, Typst, Org, Quarto, Marp, OPML, YAML, Jupyter, XML/TEI); both now describe the actual 15-format set. The privacy policy also now explicitly discloses the only third-party touchpoint — the user-initiated Tally feedback forms with their pre-filled URL parameters — instead of claiming "no network requests" unqualified.

### Changed
- **Faster page load — exporters now load on demand.** The content script core injected into every Gemini page shrank from 31 files to 8 (settings, export options, extractor, validator, download, IR filter, menu injector, orchestrator). The heavy conversion stack — the KaTeX and highlight.js vendors plus all 16 exporters — moved to `web_accessible_resources` and is dynamically imported the first time you export (memoized; retried automatically if a load fails). Menu injection, PING and quality checks work exactly as before; the first export logs its one-time load duration to the console.

### Added
- **Export profiles:** Settings → Export Options → **Profiles** saves the current setup — enabled formats, all export options and per-format overrides — under a name (up to 6, synced across devices). One click switches the whole configuration, so alternating between e.g. *Academic paper* (A4, double spacing, APA, DOI detection) and *Quick notes* (Obsidian Markdown, no TOC) no longer means retoggling half a dozen controls. The popup gains a quick profile switcher for changing setups without opening Settings; snapshots are validated on load so a stale profile can never smuggle in invalid values.
- **Export history with automatic backup:** every export now snapshots the raw report IR into `chrome.storage.local` (last 10 reports, ~2 MB each / 8 MB total, LRU-evicted; re-exporting the same report updates its slot instead of duplicating it). Settings → Tools → Re-export from JSON gains a **Recent reports** list — load any backup and convert it to any format fully offline, so a report remains recoverable even after the Gemini conversation is deleted. Entries can be removed individually or cleared entirely; nothing ever leaves the device.
- **First-run welcome:** installing the extension now opens the Settings page once with a dismissible 3-step orientation banner (open a Deep Research report → click "Share & export" or right-click → pick a format), so the new share-menu entries aren't left to be discovered by accident.
- **`README.md`:** developer-facing readme covering unpacked installation, the npm test/lint/typecheck commands and the store package build.
- **One-click diagnostics on extraction failure:** when a page yields no exportable content ("No Deep Research content found" / "Failed to read page content"), the error toast now offers a **Get diagnostics** button that downloads `gep-diagnostics.txt` on the spot — ready to attach to a bug report. The toast component gained generic action-button support alongside the existing Retry pattern.
- **Menu-injection counters in diagnostics:** the diagnostics report now includes a "Menu injection (this session)" section (menus seen / export menus matched / injected), so a silent menu breakage after a Gemini UI change becomes visible in user-submitted reports.
- **Hardened export-menu detection:** `isExportMenu` now falls back to substring matches on `data-test-id` (`*copy*`, `*export*`, `*share*`) in addition to the exact test ids, so a renamed Gemini button no longer silently disables injection.
- **Test coverage for previously untested layers** (75 new checks): `test/menu-injector.mjs` (detection incl. fallback selectors, format filtering, 12-item cap + notice, re-injection guard, no-reference fallback items, session counters), `test/background.mjs` (context menu built from defaults and stored formats, scoped `@tables` items gated by their base format, separator logic, storage-change rebuilds, context-menu clicks, all keyboard commands) and `test/content.mjs` (full-stack message-handler smoke tests: PING, EXPORT with download capture, scoped export, invalid/unknown formats, QUALITY, DIAGNOSE).

## [2.1.0] - Beta

This release ships as a **Beta** (`version_name: "2.1.0 Beta"` in the manifest; the numeric `version` stays `2.1.0`). Users are told up front: a **BETA badge** sits next to the name in both the popup and the Settings page, and a short notice warns that some exports may not be fully stable yet and asks for issue reports.

### Removed
- **Confluence Wiki (`.wiki`) and Mermaid Mindmap (`.mmd`) exports.** Confluence's wiki-markup target is effectively dead: Atlassian fully deprecated the legacy editor (the only place "Insert > Markup" accepted wiki markup) on **April 1, 2026**, and the cloud editor has no wiki-markup entry point — Confluence Cloud now ingests Markdown, which this extension already exports. Mermaid only emitted a heading/bullet mindmap of the report skeleton, a niche visual extra with no real audience for Deep Research output. Both formats and all their wiring (exporter modules, manifest entries, settings defaults, popup/options/context-menu items, and tests) were removed to keep the format set focused.

### Changed
- **Better out-of-the-box defaults for the end user.** Footnotes/sources are now included by default (`include_footnotes: true`) — a Deep Research report's citations are its core value, so the default DOCX/PDF/Markdown export no longer silently drops them. The default menu also swaps the low-value Plain Text (`.txt`) entry for **Reader HTML** (a clean, styled, self-contained offline document), so the out-of-the-box set is *Copy as Markdown · Markdown · Reader HTML · Word · PDF*. `.txt` and everything else remain one toggle away in Settings.

### Fixed
- **Empty box (□) after ∑ / ∫ in DOCX math:** large operators were emitted as an OMML `<m:nary>` whose operand slot (`<m:e>`) was left empty — because the converter lets the summand/integrand flow as the following siblings rather than binding it — so Word drew the empty naryand as a placeholder box right after the operator (visible when the `.docx` was rendered to PDF). Large operators (∑ ∏ ∫ …) now render as a scripted operator glyph instead (stacked limits via `limLow`/`limUpp`, side limits via `sSub`/`sSup`/`sSubSup`), which has no operand slot, so the box is gone and the expression after the operator still flows correctly.
- **Right-click menu ignored the selected formats:** the context menu was built once at install time from a hardcoded list, so it always showed *every* format regardless of the user's choices and never updated when they changed. It is now generated from the stored format settings (only enabled formats appear), rebuilt on install/startup and whenever the format toggles change, with separators drawn only between non-empty groups (no dangling separators).
- **EPUB math missing `properties="svg"`/`"mathml"` (OPF-014):** KaTeX formulas are re-inlined as `<svg>` (and MathML emits `<math>`) inside `chapter.xhtml`, but the OPF manifest item never declared the EPUB 3 content properties for them, so EPUBCheck rejected any math-bearing report (`ERROR(OPF-014): The property "svg" should be declared in the OPF file`). The chapter is now built before the OPF so the manifest item declares `properties="svg"`/`"mathml"` exactly when those elements are present (and stays clean when they're not).
- **Duplicate heading-anchor IDs with non-Latin headings:** body heading `id`s in HTML/Reader/PDF/EPUB were produced by calling `GEP.toc.slugify(text)` per heading with no deduplication. Arabic/CJK (and other non-Latin) headings all slugify to the `"section"` fallback, so every `<hN>` collapsed to `id="section"` — the W3C Nu Html Checker and EPUBCheck rejected the files (`Duplicate ID "section"`). The body now mirrors `toc.js`: only non-empty headings get an id, deduplicated with a `-N` suffix, so the anchors are unique *and* match the generated TOC links. Affects HTML, Reader, PDF and EPUB (all share `GEP.pdf.bodyHtml`). 
- **AsciiDoc math was dropped entirely:** the AsciiDoc exporter had no `math` block case and ignored inline `run.math`, so every formula vanished from the output (the rendered `.adoc.html` showed no equations). It now emits inline math as `stem:[…]` and display math as a `[stem]` passthrough block, and sets the `:stem: latexmath` document attribute so asciidoctor processes the math and auto-includes MathJax in standalone HTML. `]` inside inline math is escaped so it can't close the macro early.
- **Contiguous heading levels in AsciiDoc and reStructuredText:** both exporters now use the same parent-stack normalization as HTML/PDF ("nearest shallower parent + 1") instead of a flat min-based offset (AsciiDoc) / the raw IR level (RST). An in-document heading jump (e.g. h2 → h4) no longer leaks through, clearing asciidoctor's "section title out of sequence" warnings (AsciiDoc now validates clean) and docutils' "Inconsistent title style: skip from level N to M" error (RST). Regenerated the `validate/debug` `.adoc`/`.rst` fixtures from the report IR via the new `scripts/regen-adoc-rst-debug.mjs`, which also self-verifies that the rendered heading sequence never skips.
- **reStructuredText grid tables with multi-line cells:** a newline inside a cell broke the column alignment, so docutils rejected the whole table ("Blank line required after table"). Cell content is now collapsed to a single line.
- **reStructuredText stray inline markup:** a literal backtick or asterisk in the source prose (e.g. an unclosed `` ` `` carried over from the report) opened an interpreted-text/emphasis role docutils flagged as unterminated. Such characters are now backslash-escaped in non-code text.
- **reStructuredText math `\big`/`\bigg` delimiters:** docutils' math parser rejects the `\big`/`\Big`/`\bigg`/`\Bigg` delimiter-sizing macros ("Unknown LaTeX command"). They are now stripped (the bare delimiter renders fine), alongside the existing `\dots` → `\ldots` mapping.
- **Bogus code-block languages from descriptive labels:** Gemini sometimes labels a code block with a phrase ("Kod Snippeti", "INI / TOML") rather than a language token; the extractor collapsed these into invalid identifiers (`kodsnippeti`, `initoml`) that tripped syntax highlighters and RST's Pygments lookup. A fence language must be a single token, so multi-word labels are now treated as "no language" — fixing the emitted fence across Markdown, RST, AsciiDoc and Org.
- **Valid offline math in HTML/Reader/PDF/EPUB:** KaTeX renders stretchy glyphs (square-root vincula, big delimiters, arrows) as inline `<img src="data:image/svg+xml;utf8,<svg …>">`. The raw `<`/quotes left in that data URI are an illegal URL — the W3C Nu Html Checker and EPUBCheck both rejected the file — and the `<img>` had no `alt`. A shared `GEP.pdf.sanitizeMathHtml` now percent-encodes the SVG data URI and adds an empty `alt`, so math-heavy reports validate cleanly (EPUBCheck `RSC-020` and vnu "Illegal character"/"img must have alt" errors gone). Idempotent and applied at every HTML-family emit point.
- **Contiguous heading levels:** heading normalization now uses a parent stack so rendered levels never skip (previously only the shallowest level was anchored, so an in-document `h3 → h5` jump still tripped vnu's "skipping a heading level"). Each heading renders as "nearest shallower ancestor + 1".
- **Typst `partial` derivative:** `∂` now emits Typst `partial` instead of the deprecated `diff`, clearing the `typst compile` deprecation warnings.
- **Multilingual LaTeX (any engine):** the `.tex` preamble now branches on `iftex` — pdfLaTeX keeps `inputenc`/`fontenc` plus `newunicodechar` mappings for common Unicode prose symbols (sub/superscripts, `√ ∞ ∑ ∏ ∫ ∂ ∇ ≈ ≠ ≤ ≥ → ⇒ Δ Ω µ €`), while LuaLaTeX/XeLaTeX load `fontspec`. As a result a report mixing CJK, Arabic, Cyrillic and emoji no longer fatally crashes pdfLaTeX (`Unicode character … not set up`); under a Unicode engine it compiles and merely warns about glyphs the font lacks. The external validator now compiles `.tex` with `lualatex` → `xelatex` → `pdflatex` (first available) and reports missing-glyph counts as a warning instead of a failure.

### Added
- **Page & typography controls (PDF / HTML / DOCX / LaTeX):** Settings → Options → **Document Layout** adds shared layout options that flow into all four document formats through one builder (`GEP.exportOpts.build`): **paper size** (A4 / US Letter), **margins** (narrow / normal / wide), **base font size** (10 / 11 / 12 pt), **line spacing** (normal / 1.5 / double) and **font family** (sans / serif). PDF/HTML rebuild the print stylesheet (`@page size/margin`, body `font-size`/`line-height`/`font-family`); DOCX sets `w:pgSz`/`w:pgMar`/`w:rFonts`/`w:sz`/`w:line` (mm→twips, pt→half-points); LaTeX sets the `documentclass` size/paper and adds `geometry`/`setspace`/font-family directives. Defaults reproduce the previous output exactly (the lone intentional change: LaTeX's default body font is now sans to match the other formats — pick **Serif** for the classic roman look). The Reader HTML format keeps its own dedicated controls.
- **Source hygiene (dedupe / sort / DOI-ISBN):** Settings → Options → **Sources** adds an offline pre-export transform (`GEP.sourceHygiene`): **merge duplicate sources** by a normalized URL (lowercased host, dropped `www.`/fragment/tracking params, http→https) — references are remapped and renumbered 1..N; **sort the sources list** by order of appearance (default), title (alphabetical) or domain, with body references kept in lock-step; and **DOI/ISBN detection** (regex over each source's URL + title, no network) that surfaces as `doi`/`isbn` fields in the BibTeX (`doi`/`isbn`), RIS (`DO`/`SN`) and CSL-JSON (`DOI`/`ISBN`) exports. Defaults are safe (dedupe off, appearance order, detection on but purely additive).
- **Offline re-export from JSON:** Settings → Tools → **Re-export from JSON** loads a `.json` report previously exported by the extension and converts it to any other format **entirely offline** — no Gemini page required. It reuses the exact same options pipeline (layout, citations, source hygiene) and exporter stack as a live export, routing string formats and DOCX/EPUB/Vault to a download and PDF to the print dialog.
- **Multilingual & right-to-left (RTL) reports:** the extractor now records the report's language (`document.documentElement.lang`) and detects its base writing direction from the content (`GEP.extractor.detectDir`, covering Arabic, Hebrew, Persian, Urdu, Syriac, Thaana, NKo and the Arabic/Hebrew presentation forms). That `lang`/`dir` flows into every display format: HTML/Reader/PDF set `<html lang dir>` and tag each block (`<p>`, headings, `<li>`, `<th>/<td>`, `<figcaption>`) with `dir="auto"` so mixed-language reports align per block; their CSS was switched to logical properties (`padding-inline-start`, `border-inline-start`, `text-align:start`) so list/blockquote/table edges mirror correctly. EPUB sets `<dc:language>`, `xml:lang`/`lang`/`dir` on the XHTML root and `dir` on `<body>`. DOCX flips `styles.xml` document defaults to `<w:bidi/>`/`<w:rtl/>` with right alignment and emits `<w:bidiVisual/>` on tables so RTL documents open right-to-left in Word. Non-Latin (CJK, Cyrillic, Greek, …) reports were already captured; this makes them *display* correctly too. Backward compatible: IRs without `lang`/`dir` fall back to no `lang` attribute and `ltr`. **Known limit:** true RTL typesetting in LaTeX (which needs `polyglossia`+`bidi`+XeLaTeX) is out of scope for now — `.tex` still captures the Unicode text but lays it out left-to-right.
- **Report a Bug:** a "Report a bug" action in the popup and a matching row under Settings → Tools → Debug open a [Tally](https://tally.so) form, pre-filling the extension version and browser (and, from the popup, the active Gemini page URL) via query parameters so reporters don't have to dig that up. The destination lives in one place (`src/lib/links.js`); set your Tally form id there before publishing.
- **External validator `--render-out[=dir]`:** the validators already convert several formats to a viewable artifact while checking them (asciidoctor/pandoc/quarto/marp → HTML, LaTeX/typst → PDF, LibreOffice DOCX/ODT → PDF, mermaid → SVG); these were written to a temp dir and discarded. The new flag keeps them in `dir` (default `validate/render`) for manual inspection, mirroring each source file's relative path so same-kind files never collide. Markdown/Org HTML is now rendered with pandoc `--standalone` so the saved file opens as a complete document, and the AsciiDoc render carries the math (see the `:stem:` fix above).
- **HTML – Reader export (`.html`):** a new self-contained, reading-optimized HTML output aimed at end users who just want to keep and read the report. It reuses the shared body builder (so headings, TOC, footnotes and math match the other exports) but wraps it in a clean, modern, responsive shell with comfortable typography, a centered reading column, styled tables/code/blockquotes/footnotes and automatic light/dark theming via `prefers-color-scheme`. It also ships a small, progressively-enhanced reading layer: a manual **Auto/Light/Dark theme toggle** remembered in `localStorage` (applied before first paint, no flash), a **reading-progress bar** and **back-to-top** button, a sticky **side outline with active-section highlighting** (scrollspy) on wide screens, hover **anchor links** on headings, a `:target` jump highlight, balanced/pretty `text-wrap`, a `prefers-reduced-motion` guard, and a **footer crediting the extension with an estimated reading time**. Code blocks get **syntax highlighting** (highlight.js + GitHub light/dark token colors, scoped to the theme system) — the ~119 KiB highlight.js payload is inlined **only when the report actually contains code**. Fully offline — no external CSS, fonts or scripts; KaTeX styles are inlined only when the report contains math. Available in Settings (Formats → Text), the popup, the context menu, as a primary-format / per-format-override target, and in the Download-all bundle.
- **External-tool validator (`npm run validate:external`):** a `scripts/external-validate.mjs` that automates the manual checks in `Manuel-Validate/validate-yontemleri.MD` — it runs each format's real reference parser/compiler (asciidoctor, pdflatex, typst, pandoc, quarto, EPUBCheck, and python's pybtex/rispy/docutils/yaml/nbformat/ElementTree/csv/zipfile/html.parser) against the exported files and classifies every file as PASS/WARN/FAIL/SKIP. It auto-detects two layouts: a *matrix* folder (one `<fmt>/output.<ext>` per format, e.g. `Manuel-Validate`) and a *flat* folder (e.g. a debug export containing every format × citation style × markdown flavor), where it routes each file to its parser by extension and prints a per-format breakdown — full combinatorial coverage. Missing tools or files yield SKIP (never a false FAIL); `--skip-heavy` skips the browser/npx checks (Marp, Mermaid) and `--only=tex,epub` runs a subset. Exit code is non-zero only on a real FAIL.
- **Stronger validators (replace lenient checks):** the external validator now uses real validators where it previously only parsed structure — **KaTeX** renders every `$…$`/`$$…$$`/`\(…\)`/`\[…\]` span in Markdown/Org (the same engine the HTML/EPUB exporters ship), so genuinely broken math FAILs while pandoc 3.x's false "Could not convert TeX math" `\frac` warnings are dropped to PASS; **vnu.jar** (W3C Nu Html Checker) for true HTML5 validation; **LibreOffice** headless `--convert-to pdf` to prove DOCX/ODT actually open and render; **ajv** + the official `csl-data.json` schema for real CSL-JSON validation; and **jing/xmllint** RELAX NG (or a TEI namespace + `teiHeader`/`text` shape check) for TEI XML. Each degrades gracefully (SKIP or a lighter check) when its tool/resource is absent; drop `vnu.jar`/`tei_all.rng` into the target dir, `scripts/tools/` or `scripts/schemas/` to enable them.
- **Debug export condition variants:** the hidden debug export now also emits a `variants/` set (Markdown, LaTeX, HTML, Typst, RST, AsciiDoc, Org, DOCX, EPUB, ODT) with **TOC off**, **footnotes off** and **metadata on**, so the external validator covers those render paths in addition to the format × citation-style × markdown-flavor matrix. The metadata variant also emits an **Obsidian-flavored** copy (`variants/markdown-obsidian-meta.md`) so the YAML front-matter metadata path — which the GFM inline-block path doesn't exercise — gets validated too.
- **Tabbed Settings:** the Settings page is organised into **Formats**, **Options** and **Tools** tabs to reduce clutter; the Formats tab shows a live count of enabled formats, search spans every tab, and the last-used tab is remembered.
- **More citation styles:** Vancouver, Harvard, ACS and AMA join Numbered/APA/MLA/Chicago/IEEE, available in Settings, the popup and per-format overrides.
- **Per-format overrides:** Settings → Per-format Overrides lets a single format override the global Table of Contents, Footnotes and Citation Style (each control can *Inherit* the global value or be set explicitly). Applies to single exports and the Download-all bundle, and is included in backup/restore.
- **Pre-export Quality Check:** Settings → Quality Check scans the current Deep Research page locally (no network) for empty tables, heading-level jumps, unmatched footnotes, sources never referenced, duplicate source URLs, images missing alt text, `[object Object]` leaks and suspiciously short content, then downloads a `gep-quality.txt` report.
- **Code block language tags:** detected from the page (`language-…`/`lang-…`/`data-language`) and preserved in Markdown (` ```lang `), reStructuredText (`.. code-block::`), AsciiDoc (`[source,lang]`), Org (`#+BEGIN_SRC lang`) and HTML (`<code class="language-…">`).
- **Footnote/source back-links:** HTML, PDF and EPUB now give each footnote reference a unique anchor and add a return link (↩) from the source list back to the text.

### Fixed
- **Prose lost around inline math/images (all formats):** when prose immediately followed an inline formula or image, the extractor merged that text into the math/image run, where every exporter then dropped it — so a sentence like "…$x(t)$ sinyali, $T_s$ örnekleme…" collapsed to "$x(t)$$T_s$…" with the words between formulas gone. Math, image and footnote runs are now kept as self-contained inline atoms, so surrounding prose is preserved everywhere (Markdown, LaTeX, Typst, RST, Org, DOCX, HTML, EPUB, …).
- **LaTeX code blocks ran off the page:** long code lines used plain `verbatim`, which does not wrap, causing Overfull `\hbox` and text past the margin. Code now uses `fvextra`'s `Verbatim` with `breaklines`/`breakanywhere`, so even space-less tokens wrap within the text width.
- **LaTeX tables: justification warnings and split-box errors:** narrow justified `p{}` columns produced a flood of Underfull `\hbox` warnings (one word per line) and "infinite glue shrinkage" notices. Table columns are now ragged-right (`>{\raggedright\arraybackslash}p{…}`), cutting these warnings dramatically.
- **EPUB rejected by EPUBCheck (fatal):** Gemini's rendered KaTeX HTML contains `&nbsp;`, which is undeclared in XHTML and made the EPUB chapter unparseable — a single fatal error that cascaded into 19 bogus "fragment identifier not defined" errors for the table of contents. Named character entities are now converted to numeric references (`&nbsp;` → `&#160;`, etc.), so the package validates cleanly.
- **AsciiDoc heading levels out of sequence:** reports whose body starts at H2 produced `=` (title) followed directly by `===`, triggering asciidoctor "section title out of sequence" warnings. Content headings are now anchored to level 1 (`==`) so levels never skip.
- **HTML/PDF skipped a heading level (W3C):** content headings were emitted as `block.level + 1`, so reports whose body starts at H2 produced `<h1>` (title) directly followed by `<h3>` — a skipped level the W3C/Nu validator flags (it was previously masked only when the Table of Contents or Abstract added an intervening `<h2>`). Content headings are now normalized to start at `<h2>` under the title and stay contiguous (surfaced by the new vnu.jar check on the TOC-off variant).
- **reStructuredText nested lists & math:** nested list items are now indented past the parent marker and separated by a blank line (docutils no longer warns "list ends without a blank line; unexpected unindent"), and `\dots` in math is rewritten to `\ldots`, which docutils' math role understands.
- **LaTeX export failed to compile with math:** the preamble was missing `amsmath`/`amssymb`, so formulas using `\text{…}` (which Gemini emits, e.g. `\text{için}`) raised `Undefined control sequence` under `pdflatex`. Both packages are now loaded.
- **Native math in DOCX, ODT and Typst:** added a dependency-free LaTeX-math converter (`src/lib/texmath.js`) that turns the captured LaTeX into Office Math (OMML) for Word, an embedded MathML formula object for LibreOffice/ODF, and Typst math syntax. These formats previously dumped raw LaTeX text that the target apps could not render. Covers super/subscripts, fractions, roots, big operators with limits (∑ ∫ ∏ …), Greek letters, relations/operators, `\text`, function names, spacing and `\left…\right` fences, degrading gracefully to plain text on anything unrecognised.
- **Offline math rendering in HTML/PDF/EPUB:** real Gemini formulas ship only LaTeX (`data-math`, no MathML), so self-contained exports previously showed raw `\[ … \]` source instead of the equation. The extractor now also captures KaTeX's already-rendered HTML, and HTML/PDF/EPUB bundle the KaTeX stylesheet (woff2 fonts inlined as data URIs) only when the report contains math — fully offline, no network, identical to the source page.
- **Jupyter code cells leaked fences:** language-tagged opening fences (` ```python `) were not recognised by the `.ipynb` splitter, so fences and prose ended up in the wrong cells. Opening fences with a language label are now detected correctly.
- **Code language on real Gemini pages:** language tags are now read from Gemini's `<code-block>` header label (e.g. *Python*, *JavaScript*, *C*), not just `language-…` classes — real reports previously exported bare code fences.
- **Math on real Gemini pages:** formulas are now captured from Gemini's `.math-inline` / `.math-block` wrappers via the `data-math` LaTeX attribute (KaTeX without MathML/annotation). Previously these were dropped entirely, leaving gaps in the prose.

### Changed
- Injected toast notifications are now isolated in a Shadow DOM so Gemini's global styles cannot affect them.
- Improved `@media print` rules for PDF/HTML: A4 page margins, orphan/widow control, heading `break-after` avoidance, and table/figure/code block page-break protection.
- The most recently extracted IR is cached in memory so exporting the same report in multiple formats no longer re-scans the DOM each time.

### Known limitations
- Typst math is best-effort: LaTeX syntax is emitted between `$…$` and may need manual adjustment (Typst's math syntax differs from LaTeX).
- DOCX renders math as italic LaTeX text (no OMML); HTML/PDF/EPUB render true MathML.
- ODT supports core blocks (headings, paragraphs, lists, tables, blockquotes, code, math); advanced styling is lighter than DOCX.
